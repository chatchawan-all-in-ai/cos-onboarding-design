Decision notes

⭐ Recommendation: Variant B — Conversational Chat (bot-first)

เป้าหมายคือ "ผู้ใช้เห็นบอทตอบ" เร็วที่สุด. Variant B บรรลุเป้าได้ดีที่สุดเพราะ: (1) ผู้ใช้เห็น mini-preview บอทตัวเองก่อนถูกขอ LINE token — aha moment เกิดก่อน friction. (2) "เชื่อม LINE ทีหลัง" skip ช่วยให้ activation path ไม่บล็อก. (3) AIda persona บอก story ให้เชื่อมโยงกับ product จริง (AI Chat CRM). (4) Non-technical SME เข้าใจ conversational UI ได้ดีกว่า config form.

ข้อแลกเปลี่ยน: ต้องการ model change (decouple bot จาก LINE OA) — ถ้ายังไม่พร้อม ให้ implement Variant A ก่อนเป็น interim แล้ว migrate ไป B เมื่อ model พร้อม.

Channel-first vs. Bot-first: the open decision

Channel-first (Variant A): ตรงกับ data model ปัจจุบัน (bot 1:1 กับ LINE OA). Build cost ต่ำ. แต่ LINE token friction เกิด step 2 ก่อน activation — ผู้ใช้ต้องออกไป Developer Console ก่อนจะ "รู้สึก" ว่า setup สำเร็จ.

Bot-first (B หรือ C): ต้องการ API ใหม่ (POST /api/v1/bots) และ decouple bot จาก LINE OA. Build cost สูงกว่า แต่ activation flow ดีกว่าอย่างมีนัยสำคัญ — LINE token กลายเป็น "optional next step" ไม่ใช่ "ด่านบังคับ".

ข้อแนะนำ: commit to bot-first if the sprint allows it. หากไม่ได้ ให้ทำ Variant A ก่อนแต่ต้องย้าย LINE connect ไปหลัง template selection ให้มากที่สุด

Visual QA Notes

A: Step navigation tabs (demo only) อาจ confuse ถ้าใส่ production — ให้ลบออก. Webhook URL display ต้องการ real slug generation ใน backend. LINE screenshot placeholder ต้องการ real screenshot.

B: Bottom bar fixed position อาจ overlap content บน mobile — ต้องทดสอบ iOS Safari safe area. Demo nav (prototype only) ต้องลบ production. "เชื่อม LINE ทีหลัง" button ต้องเชื่อม onboarding_state tracking จริง.

C: Split layout collapse ที่ <900px — ต้องทดสอบ responsive breakpoint. Live phone mockup ต้องการ streaming API หรือ mock response service. Dark mode by default อาจไม่ตรงกับ design system ที่เหลือ (light by default).

Variant C — When to use

Variant C เหมาะสำหรับ "re-configure" flow (ผู้ใช้กลับมาแก้ไขบอทที่มีอยู่แล้ว) มากกว่า first-time onboarding. ลักษณะ split-screen console ตรงกับ Bot Builder page ที่มีอยู่แล้ว (/dashboard/bots/[id]) — อาจ merge C เข้ากับ Bot Builder แทนที่จะเป็น onboarding wizard.

ถ้าต้องการ implement C ให้ใช้เป็น "expert mode" toggle ใน Variant A หรือ B — ไม่ใช่ default path สำหรับ new users.
